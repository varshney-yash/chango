About Django Channels

Django Channels is a project that extends the capabilities of Django beyond HTTP to handle WebSockets, chat protocols, IoT protocols, and more. It's built on a Python specification called ASGI (Asynchronous Server Gateway Interface) and provides the choice to handle other connections in either a synchronous or asynchronous style.


Unlike the core Django framework, Channels isn’t underpinned by WSGI; instead, it’s built on a newer Python spec called ASGI which supports both synchronous and asynchronous code. Channels extends Django’s capabilities, making it possible to handle not only HTTP but also WebSockets, MQTT (IoT), chatbots, radios, and other real-time applications.


Django Channels offers a certain degree of flexibility, allowing you to configure your new Django project in a way that makes the most sense for your use case. It has multi-protocol support, enabling the use of WebSockets, HTTP (including HTTP/2, long-polling, and Server-Sent Events), and other options like chat and IoT protocols.


Channels supports a 1:1 messaging pattern, but it also allows you to broadcast events to multiple consumers via channels. Channels provides different routing classes which allow you to combine and stack consumers, similar to Django's URLs.


Channels is comprised of several packages: channels, the Django integration layer; daphne, the HTTP and Websocket termination server; asgiref, the base ASGI library; and channels_redis, the Redis channel layer backend (optional). Channels builds upon the native ASGI support in Django, giving you the choice to handle other connections in either a synchronous or asynchronous style.


Channels is a powerful tool for building real-time applications and can be used to power a wide range of use cases, from chat applications to IoT protocols.
